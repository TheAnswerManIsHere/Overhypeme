"All" button as default? No. "All" would re-introduce the exact conflation problem you're solving. If I'm looking at "All" and I see a meme, I still don't know if it's mine or someone else's without inspecting it. The default should be "Community" (others' public) — that's the highest-value view for most visits. It answers "what has the community made for this fact?" which is the primary discovery use case. Your own memes are a secondary, intentional lookup. An "All" tab adds UI clutter without adding information — it's just the union of the other three, and the user can mentally reconstruct that by clicking through each.
Unauthenticated users — hide or disable? Hide entirely. Disabled buttons with tooltips are a UX antipattern for this context. A disabled "My Private" button communicates nothing useful — it just raises a question the user didn't have. The principle here: don't show controls that advertise features the user can't use unless those controls serve as a conversion funnel. And "My Private memes" is not a compelling conversion hook — your login CTA elsewhere on the page handles that. Unauthenticated users see one button (Community), which is actually cleaner and less cognitively loaded. When they authenticate, the "My Public" and "My Private" tabs appear — a nice progressive disclosure moment that signals "you now have new capabilities."
Rename to "Community"? Yes, strongly. "Others' Public" is descriptive but clinical — it reads like a database field name, not a UI label. "Community" is shorter, warmer, and immediately conveys the social context. It also sets up a clean trio: Community / My Public / My Private. The semantic contrast between "Community" (not mine) and "My ___" (mine) is immediately intuitive without any explanation needed.
One thing worth flagging: you'll want to think about what happens when a filter returns zero results. "My Private" with no memes is a great place to put a CTA to the meme generator — "Create your first private meme for this fact." That turns an empty state into a conversion moment rather than a dead end.
