# Feature Request: Add Style Modifier Dropdown to AI Image Generator

## Overview

Add a “Style” dropdown selector to the AI image generation system that appears in both the **generic image generator** (text-to-image) and the **reference-based image generator** (image edit/transform). The dropdown lets users choose a visual style that gets appended to their prompt before it’s sent to the OpenAI API.

The OpenAI GPT Image models (gpt-image-1, gpt-image-1.5) do **not** have a native `style` API parameter — style is controlled entirely through prompt text. So the implementation is a prompt-concatenation pattern: whatever base prompt exists gets the selected style’s prompt suffix appended to it before the API call.

-----

## UI Implementation

### Dropdown Component

Add a styled `<select>` dropdown (or a custom dropdown component consistent with the existing site design — dark background, fire-orange `#ff6b35` accent, DM Sans font) in both image generation forms:

1. **Generic / Text-to-Image generator** — place the Style dropdown near the prompt input field, before the “Generate” button.
1. **Reference-based / Image Edit generator** — same placement, near the prompt and image upload controls.

The dropdown should:

- Default to “Default (No Style)” which is the first item in the array.
- Show all style labels in the dropdown list.
- Store the selected style `id` in component state.

### Prompt Assembly Logic

When the user submits a generation request, assemble the final prompt as follows:

```typescript
function buildFinalPrompt(
  basePrompt: string,
  selectedStyleId: string,
  hasReferenceImage: boolean
): string {
  const style = IMAGE_STYLES.find((s) => s.id === selectedStyleId);

  if (!style || style.id === "none") {
    return basePrompt;
  }

  const suffix = hasReferenceImage
    ? style.promptSuffixReference
    : style.promptSuffix;

  return `${basePrompt.trim()} ${suffix}`;
}
```

This assembled prompt is what gets sent to the OpenAI API in the `prompt` field. No other API parameters need to change — the style is entirely prompt-driven.

### Important: Do NOT use the API’s `style` parameter

The `style` parameter in the OpenAI API only exists for DALL-E 3 (with values `vivid` or `natural`). The GPT Image models (gpt-image-1, gpt-image-1.5) do not support it. All style control is done through prompt text, which is what this implementation does.

-----

## Styling Guidelines

Match the existing site aesthetic:

- Dropdown background: dark (`#1a1a2e` or similar from existing palette)
- Text color: light/white
- Accent/focus border: `#ff6b35` (fire-orange)
- Font: DM Sans (body font used across the site)
- Border radius and spacing should match existing form controls
- On mobile, the dropdown should be full-width within its container

|What                    |Where                                                                                                                                |
|------------------------|-------------------------------------------------------------------------------------------------------------------------------------|
|Styles config/data      |New file in config or constants directory (e.g. `imageStyles.ts`)                                                                    |
|Dropdown component      |Either inline in existing generator forms, or as a shared `<StyleSelector>` component if both generators exist as separate components|
|Prompt assembly function|Utility/helper file or co-located with the API call logic                                                                            |
|No new API routes needed|This is purely frontend prompt manipulation before the existing API call                                                             |
